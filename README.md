# MANTIS Terminal Simulation

A fake SSH terminal honeypot simulator.

This project runs an SSH server (default port `2222`) and simulates terminal behavior with an LLM-backed planner/response pipeline.

## 1) Prerequisites

- Windows 10/11 (PowerShell)
- Conda (Miniconda or Anaconda)
- OpenSSH client (usually built in on Windows)
- Python is managed by conda environment
- Optional: Docker Desktop (for SFTP upload mirroring to container)

## 2) Create Conda Environment (Windows PowerShell)

Run from project root:

```powershell
# If conda activate does not work yet in PowerShell, run once then reopen terminal
# conda init powershell

conda create -n mantis-sim python=3.10 -y
conda activate mantis-sim
```

## 3) Install Dependencies

```powershell
python -m pip install --upgrade pip setuptools wheel
pip install -r requirements.txt
```

If `torch` install is slow or fails from pip, you can install it with conda first:

```powershell
conda install -y -c pytorch pytorch cpuonly
pip install -r requirements.txt
```

## 4) Configure API Key

Set your OpenAI API key in the current shell session:

```powershell
$env:OPENAI_API_KEY="your_openai_api_key"
```

## 5) (Optional) Start Docker SFTP Test Container

The repo includes `docker-compose.yml` with container name `terminal_sftp` mapped to host port `8022`.

```powershell
docker compose up -d
```

## 6) Start the Simulator

```powershell
python .\Terminal_simulation.py
```

Expected startup log:

```text
[*] Fake SSH server listening on 0.0.0.0:2222
```

Note: Host key files are auto-generated by the code if missing (`ssh_host_rsa_key`, optional ed25519), so manual `ssh-keygen` is not required.

## 7) Test SSH Login

Open a new PowerShell window:

```powershell
ssh root@127.0.0.1 -p 2222
```

Password auth is accepted by the simulator (password content is ignored by current implementation).

## 8) SCP/SFTP Notes

- Simulator SSH/SFTP port: `2222`
- Docker test SFTP container port: `8022`
- Uploaded files are staged under local `scp_root/`
- Then copied to Docker container/path:
  - `DOCKER_UPLOAD_CONTAINER` (default: `terminal_sftp`)
  - `DOCKER_UPLOAD_PATH` (default: `/home/sftpuser/upload`)
- Upload audit is appended to `upload_audit.jsonl`
- Uploaded files are forced to non-executable mode (`0644`)
- If AV scan marks a file infected, it is moved to `scp_root/_quarantine`

## 9) Common Windows Fixes

### A) Host key changed warning

If you see:

`WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED!`

Run:

```powershell
ssh-keygen -R "[127.0.0.1]:2222"
ssh root@127.0.0.1 -p 2222
```

### B) `WinError 10054` during connection

Usually means the client closed the connection early during handshake/auth.

Check:

- You are connecting to the correct port (`2222`)
- Server process is still running in the other terminal
- `OPENAI_API_KEY` is set in the server terminal session

### C) `stragetic_agent import failed`

Install missing runtime packages:

```powershell
pip install -r requirements.txt
```

## 10) Stop and Cleanup

Stop server: `Ctrl + C` in the server terminal.

Optional Docker cleanup:

```powershell
docker compose down
```

Deactivate conda env:

```powershell
conda deactivate
```
